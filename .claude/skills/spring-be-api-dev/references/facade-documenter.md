# Facade 문서화 가이드

---

Facade 계층 CLAUDE.md 문서 생성/리팩토링 가이드

## 핵심 원칙

---

- **기능 중심**: "무엇을 하는가"에 집중
- **간결함**: 핵심 정보만, 불필요한 수식어 제거
- **맥락 보존**: 이해에 필요한 정보는 생략 금지

## 포함 대상

---

- Facade 기능 설명 (1-2줄)
- 조율 플로우 (2-4단계)
- 핵심 비즈니스 흐름

## 제거 대상

---

- 구현 예시 코드
- 조합하는 인터페이스 상세 나열
- 메서드 시그니처
- 설계 포인트/확장 고려사항

## 문서 템플릿

---

```markdown
# {Domain} Facade

---

{도메인} 관련 Aggregate 조율

## {Facade명}

---

### 기능

---

{Facade 기능 설명 1-2줄}

### 플로우

---

1. {단계 1}
2. {단계 2}
3. {단계 3}
```

## 변환 예시

---

### ❌ 장황한 문서

```markdown
### AuthFacade 구현

AuthFacade는 인증과 관련된 여러 Aggregate를 조율하는 핵심 Facade입니다.

#### 의존성
@Component
class AuthFacade(...)

#### 로그인 플로우
1. 전화번호로 회원 조회
2. 회원이 없으면 예외 발생
3. 인증 코드 검증
4. 검증 실패시 예외 발생
5. 토큰 생성
6. 토큰 반환
```

### ✅ 간결한 문서

```markdown
## AuthFacade

---

### 기능

---

회원 인증 및 토큰 발급

### 플로우

---

1. 전화번호로 회원 조회
2. 인증 코드 검증
3. 토큰 발급
```
